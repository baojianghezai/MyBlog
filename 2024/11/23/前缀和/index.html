前缀和

2024/11/23 算法

前期最像算法的“基础算法”

首先看看怎么来的前缀和算法

题目传送门

（经典例题区间和）

学前缀和之前：

#include <iostream>
using namespace std;
const int N = 1e5+10;
int a[N];

int main()
{
  int n;
  cin>>n;
  for( int i = 1; i <= n; i++)
  {
    cin>>a[i];
  }
  int m;
  cin>>m;
  while( m-- )
  {
    int l,r;
    cin>>l>>r;
    int su = 0;
    for( int i = l ; i <= r ; i++ )
      su += a[i];
    cout<<su<<endl;
  }
}

然后浅浅看一眼时间复杂度

对于所有测试数据，n,m≤10^5,ai​≤10^4

时间复杂度是O(m*(r-l))=O(m*n)

所以我们取极限水准：m=1e5,n=1e5

算一下运算次数：10的10次方…….

正常来说算法题目使用c++一秒的运算次数在10的8次方左右

所以这么做的话肯定是完成不了我们的题目，由此我们就需要去寻找一个更好的计算方法→前缀和算法

前缀和思想：已知一个固定的序列，我想要求出从某个下标加到另一个下标，那么就可以使用累加的公式稍微变形

公式推导过程：

b[i] = a[1] + a[2] +......+ a[i]  //前缀和定义     .1
b[j] = a[1] + a[2] +......+ a[i] + a[i+1] +......+ a[j]    //.2
//假设现在我要求a[i] 加到 a[j] 的和(j>i)
将1式和2式上下相减可得
b[j] - b[i] = a[i+1] + a[i+2] +......+ a[j]
//发现等号右边加上个a[i]就是我们需要的东西了，所以等式两边同时加上a[i]
b[j] - b[i] + a[i] = a[i] + a[i+1] + a[i+2] +......+ a[j]
//所以右边那一大串就可以简化为
b[j] - b[i] + a[i]
//再根据b[i]定义
b[i] = a[1] + a[2] +......+ a[i]
//代换一下
b[j] - (a[1]+a[2]+....+a[i]) + a[i]
=b[j] - (a[1]+a[2]+......+a[i-1])
=b[j] - b[i-1]

至此，前缀和公式推导完毕

接下来就是补充细节问题

  1. b[i]如果只用公式求，那么时间复杂度会非常大,运算次数：1+2+3+4+….+n,但是我们可以运用一下递推：b[i] = b[i-1] + a[i],这样时间花费就会降到n次

  2. 根据递推公式和题目要求，如果原数组用不到的话，可以将a数组直接变成一个前缀和数组:a[i] = a[i-1] + a[i]

最终代码

#include <iostream>
using namespace std;

const int N = 1e5+10;
int a[N];

int main()
{
    int n,m,l,r;
    cin>>n;
    for( int i = 1 ; i <= n ; i++ )
    {
        cin>>a[i];
        a[i] = a[i-1] + a[i];
    }
    cin>>m;
    while( m-- )
    {
        cin>>l>>r;
        cout<< a[r] - a[l-1] <<endl;
    }
}
