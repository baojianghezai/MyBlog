前期最像算法的“基础算法”

首先看看怎么来的前缀和算法

题目传送门

（经典例题区间和）

学前缀和之前：

#include <iostream>
using namespace std;
const int N = 1e5+10;
int a[N];

int main()
{
  int n;
  cin>>n;
  for( int i = 1; i <= n; i++)
  {
    cin>>a[i];
  }
  int m;
  cin>>m;
  while( m-- )
  {
    int l,r;
    cin>>l>>r;
    int su = 0;
    for( int i = l ; i <= r ; i++ )
      su += a[i];
    cout<<su<<endl;
  }
}

然后浅浅看一眼时间复杂度

对于所有测试数据，n,m≤10^5,ai​≤10^4

时间复杂度是O(m*(r-l))=O(m*n)

所以我们取极限水准：m=1e5,n=1e5

算一下运算次数：10的10次方…….

正常来说算法题目使用c++一秒的运算次数在10的8次方左右

所以这么做的话肯定是完成不了我们的题目，由此我们就需要去寻找一个更好的计算方法→前缀和算法

前缀和思想：已知一个固定的序列，我想要求出从某个下标加到另一个下标，那么就可以使用累加的公式稍微变形

公式推导过程：

b[i] = a[1] + a[2] +......+ a[i]  //前缀和定义     .1
b[j] = a[1] + a[2] +......+ a[i] + a[i+1] +......+ a[j]    //.2
//假设现在我要求a[i] 加到 a[j] 的和(j>i)
将1式和2式上下相减可得
b[j] - b[i] = a[i+1] + a[i+2] +......+ a[j]
//发现等号右边加上个a[i]就是我们需要的东西了，所以等式两边同时加上a[i]
b[j] - b[i] + a[i] = a[i] + a[i+1] + a[i+2] +......+ a[j]
//所以右边那一大串就可以简化为
b[j] - b[i] + a[i]
//再根据b[i]定义
b[i] = a[1] + a[2] +......+ a[i]
//代换一下
b[j] - (a[1]+a[2]+....+a[i]) + a[i]
=b[j] - (a[1]+a[2]+......+a[i-1])
=b[j] - b[i-1]

至此，前缀和公式推导完毕

接下来就是补充细节问题

  1. b[i]如果只用公式求，那么时间复杂度会非常大,运算次数：1+2+3+4+….+n,但是我们可以运用一下递推：b[i] = b[i-1] + a[i],这样时间花费就会降到n次

  2. 根据递推公式和题目要求，如果原数组用不到的话，可以将a数组直接变成一个前缀和数组:a[i] = a[i-1] + a[i]

最终代码

#include <iostream>
using namespace std;

const int N = 1e5+10;
int a[N];

int main()
{
    int n,m,l,r;
    cin>>n;
    for( int i = 1 ; i <= n ; i++ )
    {
        cin>>a[i];
        a[i] = a[i-1] + a[i];
    }
    cin>>m;
    while( m-- )
    {
        cin>>l>>r;
        cout<< a[r] - a[l-1] <<endl;
    }
}

差分经典模板题

还是看一下没学差分之前，暴力解题的代码

#include <iostream>
using namespace std;
const int N = 5e6+10;//根据题目给出的n值确定数组大小
int a[N];            //确定数组大小
int main()
{
    int n,m;
    cin>>n>>m;
    for( int i = 1 ; i <= n ; i++ )
    {
        cin>>a[i];
    }
    while( m-- )     //m次循环的快捷写法
    {
        int x,y,z;
        cin>>x>>y>>z;
        for( int i = x ; i <= y ; i++ )
        {
            a[i] += z;   //从第x到第y个学生的成绩加z
        }
    }
    int mi = 0x3f3f3f3f;
    for( int i = 1 ; i <= n ; i++ )
    {
        mi = min(mi,a[i]);     //寻找最小值
    }
    cout<<mi;
}

再简单分析一下时间复杂度

while( m-- )     //m次循环的快捷写法
    {
        int x,y,z;
        cin>>x>>y>>z;
        for( int i = x ; i <= y ; i++ )
        {
            a[i] += z;   //从第x到第y个学生的成绩加z
        }

很明显这段代码就是我们运算次数最多的，看一下数据量

可以发现n的极限为510的6次方，所以for循环内最大就是510的6次方

再结合m次循环，可以发现是一个O(n^2)的时间复杂度，很明显会超时

也就是这样

那怎么去优化我们的过程呢

分析整个代码，我们可以发现，while(m--)这个循环是题目要求所必须的，所以只能对while里面的for循环下手

这里就可以从伟大的数学家那里窃取亿点点思路了

这里借鉴网上和我自己理解，写出俩差分的思路

由此我们可以考虑构建一个差分数组，让b[i]=a[i]-a[i-1] (i>0)

通过改变b[i]的值来进行[i]后面元素统一修改，再通过改变结束位置的值来进行加减相互抵消，达成区域快速加减的目的，最后通过前缀和得到原数组

代码如下

#include <iostream>
using namespace std;
const int N = 5e6+10;
int a[N],b[N];
int main()
{
    int n,m;
    cin>>n>>m;
    for( int i = 1 ; i <= n ; i++ )
    {
        cin>>a[i];
        b[i] = a[i] - a[i-1];
    }
    while( m-- )
    {
        int x,y,z;
        cin>>x>>y>>z;
        b[x] += z;
        b[y+1] -= z;
    }

    for( int i = 1 ; i <= n ; i++ )
    {
        a[i] = a[i-1] + b[i];
    }

    int mi = 0x3f3f3f3f;
    for( int i = 1 ; i <= n ; i++ )
    {
        mi = min(mi,a[i]);
    }
    cout<<mi;
}
